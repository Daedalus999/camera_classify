# ClimbSnap — 攀岩照片/视频分类管理 App

## 产品需求文档 (PRD)

### 1. 产品概述

ClimbSnap 是一款面向攀岩爱好者的照片与视频分类管理应用。用户可以按日期和攀岩馆（地点）对拍摄的照片和视频进行分类整理，为每条记录添加标题和完成状态标记，并在应用内流畅浏览，支持视频倍速播放。所有媒体文件始终存储在手机本地，应用仅维护分类索引，不进行任何云端上传。

### 2. 目标用户

- 攀岩爱好者，需要按训练日期和场馆整理攀岩照片/视频
- 运动记录者，希望通过日历和月度统计快速回顾训练内容
- 对隐私敏感的用户，不希望媒体文件上传至云端

### 3. 核心功能

#### 3.1 主页（训练统计）

| 项目 | 说明 |
|------|------|
| 训练概览 | 显示总训练天数和活跃月份数 |
| 月度列表 | 按月份倒序展示，每月显示训练天数 |
| 月度详情 | 点击月份进入月度详情页，按日期分组展示该月所有媒体 |
| 日期跳转 | 月度详情中点击日期头可跳转到日期详情页 |

#### 3.2 日历视图

| 项目 | 说明 |
|------|------|
| 月历展示 | 以月为单位展示日历，有媒体记录的日期显示彩色标记点（颜色取自关联地点） |
| 日期选择 | 点击日期进入该日详情页，查看所有上传的照片和视频 |
| 快速导航 | 支持左右滑动切换月份，点击"今天"快速回到当前日期 |
| 快速上传 | 日历页右下角浮动按钮可快速进入上传页 |

#### 3.3 媒体上传与分类

| 项目 | 说明 |
|------|------|
| 选择媒体 | 从手机相册选择照片或视频（原生端使用 expo-image-picker，Web 端使用 file input） |
| 指定日期 | 上传时选择关联日期（默认当天或从日历页传入的日期），支持日历选择器切换 |
| 指定地点 | 上传时选择已有地点或新建地点 |
| 添加标题 | 可选，最多 10 个字符，用于标记路线名称等 |
| 标记状态 | 可选，支持"无状态"、"成功"（完攀）、"尝试"三种状态 |
| 批量上传 | 支持一次选择多张照片/多个视频，共享同一标题、状态和地点 |
| 本地存储 | 媒体文件不复制不移动，仅记录文件路径引用 |

#### 3.4 媒体浏览

| 项目 | 说明 |
|------|------|
| 网格浏览 | 照片和视频以 3 列缩略图网格展示 |
| 同名分组 | 标题相同的媒体（2 个及以上）自动聚合为可展开/折叠的子集合 |
| 状态徽章 | 缩略图左上角显示"成功"（绿色）或"尝试"（紫色）状态徽章 |
| 地点色点 | 缩略图右上角显示地点颜色标记 |
| 标题栏 | 有标题的缩略图底部显示标题文字 |
| 全屏查看 | 点击缩略图进入全屏查看，照片支持缩放 |
| 视频播放 | 内置视频播放器，支持播放/暂停/进度条/前进后退 10 秒 |
| 倍速播放 | 视频支持 0.5x / 1x / 1.5x / 2x / 3x 倍速切换 |
| 详情面板 | 全屏模式下可查看媒体详情（标题、状态、类型、日期、地点、时长、尺寸） |

#### 3.5 媒体编辑

| 项目 | 说明 |
|------|------|
| 编辑入口 | 全屏查看页顶部工具栏的编辑按钮（铅笔图标） |
| 编辑标题 | 修改或清除标题 |
| 编辑状态 | 切换"无状态"/"成功"/"尝试" |
| 编辑地点 | 更换关联地点或取消关联，支持新建地点 |
| 删除记录 | 仅删除分类索引，不删除手机相册中的原始文件 |
| 分组联动 | 修改标题后，该媒体自动脱离原同名子集合 |

#### 3.6 地点管理

| 项目 | 说明 |
|------|------|
| 自定义地点 | 用户可自由创建地点名称，自动分配颜色（10 种预设色循环） |
| 地点列表 | 查看所有已创建的地点，显示颜色标记和关联媒体数量 |
| 地点详情 | 点击地点查看该地点所有媒体，显示照片/视频/天数统计 |
| 编辑地点 | 支持编辑地点名称 |
| 删除地点 | 删除地点后，已关联的媒体不会被删除，但会取消地点关联 |
| 快速创建 | 在地点列表底部输入框或上传/编辑页的地点选择器中均可新建地点 |

#### 3.7 设置

| 项目 | 说明 |
|------|------|
| 应用信息 | 显示应用名称、版本号、功能描述 |
| 存储说明 | 说明所有媒体始终保存在手机本地，应用不进行云端上传 |
| 清除数据 | 清除所有分类索引数据（带二次确认弹窗和警告图标），不删除原始文件 |
| 操作反馈 | 清除成功/失败均有结果提示弹窗（含状态图标） |

### 4. 非功能需求

| 项目 | 说明 |
|------|------|
| 隐私 | 所有数据存储在本地，无网络请求，无云同步 |
| 性能 | 大量媒体时列表流畅滚动，使用虚拟化列表和手动行布局 |
| 平台 | 支持 iOS、Android 和 Web（Web 使用 localStorage 替代 SQLite） |
| 离线 | 完全离线可用 |
| 数据安全 | 重复清除数据不会报错，数据库操作使用事务保证一致性 |

### 5. 页面结构

```
Tab 导航（4 个标签）
├── 主页 (home)
│   ├── 训练统计概览（总天数、活跃月份）
│   └── 月份列表 → 月度详情页 (month/[month])
│       └── 按日期分组的媒体网格 → 日期详情页
├── 日历 (index)
│   ├── 月历视图（带标记点）
│   └── 日期详情页 (day/[date])
│       ├── 按地点分组的标签
│       ├── 媒体网格（含同名分组）
│       └── 浮动上传按钮 → 上传页
├── 地点 (locations)
│   ├── 地点列表（带颜色、计数、编辑/删除）
│   ├── 底部新建输入框
│   └── 地点详情页 (location/[id])
│       └── 该地点所有媒体网格
└── 设置 (settings)
    ├── 应用信息
    ├── 存储说明
    └── 清除数据

独立页面
├── 上传页 (upload) — Modal 形式
│   ├── 日期选择器
│   ├── 地点选择器
│   ├── 标题输入
│   ├── 状态选择
│   └── 媒体选择网格
└── 媒体查看页 (media/[id]) — 全屏 Modal
    ├── 全屏照片/视频
    ├── 顶部工具栏（关闭、编辑、详情、删除）
    ├── 详情面板（底部滑出）
    └── 编辑面板（底部滑出 Sheet）
```

### 6. 交互流程

#### 上传流程
```
日历页/日期详情页 → 点击"+"按钮 → 选择日期 → 选择地点（或新建）→ 输入标题（可选）→ 选择状态（可选）→ 从相册选择照片/视频 → 保存
```

#### 浏览流程
```
日历页 → 选择日期 → 查看该日媒体网格 → 展开同名分组（如有）→ 点击进入全屏 → 查看详情 / 播放视频（可调倍速）
```

#### 编辑流程
```
全屏查看页 → 点击编辑按钮 → 修改标题/状态/地点 → 保存 → 返回列表自动刷新
```

#### 月度回顾流程
```
主页 → 点击月份卡片 → 查看该月所有日期的媒体 → 点击日期头跳转到日期详情 → 查看/编辑具体媒体
```
